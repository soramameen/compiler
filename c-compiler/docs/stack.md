# pushとpopについて考える。

コード生成を考える上でpushとpopは何回も使うであろうことが考えられるため、切り出す。

## push

一番上に格納する。今回はスタックは下側に(アドレスが高い方から小さい方に向かって)伸びていることに注意する。
スタックを伸ばして、指定したレジスタの中身を現在のスタックポインタから-4した場所に格納する。そしてスタックポインタをそこまで下げる。
pushはこれの繰り返しである。

## pop

一番上のデータを取り出す。
スタックの一番上のものを指定したレジスタに格納して(読み出して)スタックポインタをあげる。次またpushされた際には、もともと合ったデータは上書きされるため、解放する必要はない(?)
